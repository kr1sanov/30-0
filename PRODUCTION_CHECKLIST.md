## 🔐 Проверки безопасности

### HTTPS и транспорт

- [ ] **HTTPS работает** — сайт доступен только по HTTPS, HTTP редиректит на HTTPS
- [ ] **SSL-сертификат валиден** — `curl -vI https://30-0.рф 2>&1 | grep "expire date"` показывает корректную дату
- [ ] **HSTS включён** — заголовок `Strict-Transport-Security` присутствует
- [ ] **TLS 1.3 поддерживается** — проверьте через [SSL Labs](https://www.ssllabs.com/ssltest/)

### Заголовки безопасности

- [ ] **X-Frame-Options** — установлен в `SAMEORIGIN` (защита от clickjacking)
- [ ] **X-Content-Type-Options** — установлен в `nosniff`
- [ ] **X-XSS-Protection** — установлен в `1; mode=block`
- [ ] **Referrer-Policy** — установлен в `strict-origin-when-cross-origin`
- [ ] **Content-Security-Policy** — настроен, запрещает инъекции скриптов (разрешены Yandex.Metrika и домен приложения)

Проверка заголовков:
```bash
curl -sI https://30-0.рф | grep -E "X-Frame|X-Content|X-XSS|Referrer|Content-Security|Strict-Transport"
```

### Секреты и ключи

- [ ] **Нет секретов в коде** — в репозитории нет хардкод-токенов, паролей, ключей
- [ ] **`.env` в `.gitignore`** — файл `.env` исключён из Git
- [ ] **`.env.example` актуален** — шаблон содержит все необходимые переменные с описаниями
- [ ] **GitHub Secrets настроены** — все секреты для CI/CD добавлены в настройки репозитория
- [ ] **NEXTAUTH_SECRET сгенерирован** — используется криптографически стойкая случайная строка
- [ ] **DATABASE_URL не содержит пароль в открытом виде в логах** — Prisma не логирует credentials

### Дополнительные проверки

- [ ] **Нет открытых отладочных эндпоинтов** — `/api/debug`, `/api/test` и подобные недоступны или защищены

## 🚨 Экстренные процедуры

### Быстрый откат

```bash
# 1. SSH на сервер
ssh -p 2222 root@123.45.67.89

# 2. Перейдите в проект
cd /var/www/30-0.рф

# 3. Посмотрите историю
git log --oneline -5

# 4. Откатитесь к предыдущему коммиту
git reset --hard <РАБОЧИЙ_КОММИТ>

# 5. Пересоберите и запустите
bun install
bun run build
pm2 restart all

# 6. Проверьте
curl -f http://localhost:3000/api/health
```

### Перезапуск PM2

```bash
# Мягкий перезапуск (без даунтайма, если возможно)
pm2 reload all

# Жёсткий перезапуск (с短暂 даунтаймом)
pm2 restart all

# Перезапуск конкретного процесса
pm2 restart 30-0-app
pm2 restart 30-0-ws

# Статус процессов
pm2 status

# Логи
pm2 logs --lines 50
```

### Восстановление базы данных

```bash
# 1. Найдите последний бэкап
ls -la /var/backups/game_db_*.sql

# 2. Остановите приложение
pm2 stop all

# 3. Восстановите базу
mysql -u game_user -p game_db < /var/backups/game_db_20250101_120000.sql

# 4. Запустите приложение
pm2 start all

# 5. Проверьте
curl -f http://localhost:3000/api/health
```

### Экстренное создание бэкапа

```bash
mysqldump -u game_user -p game_db > /var/backups/emergency_$(date +%Y%m%d_%H%M%S).sql
```

### Если закончилось место на диске

```bash
# Проверьте свободное место
df -h

# Очистите старые логи PM2
pm2 flush

# Очистите старые бэкапы (старше 7 дней)
find /var/backups/ -name "game_db_*.sql" -mtime +7 -delete

# Очистите кэш Next.js
rm -rf /var/www/30-0.рф/.next/cache

# Очистите старые логи Nginx
sudo truncate -s 0 /var/log/nginx/access.log
sudo truncate -s 0 /var/log/nginx/error.log
```

### Если сервер недоступен

1. Зайдите в панель управления Jino
2. Проверьте статус VPS — запущен ли сервер
3. При необходимости перезапустите VPS через панель
4. После перезапуска проверьте, что PM2 запустился автоматически:
   ```bash
   pm2 resurrect
   pm2 status
   ```
5. Проверьте Nginx:
   ```bash
   sudo systemctl status nginx
   sudo systemctl start nginx  # если не запущен
   ```

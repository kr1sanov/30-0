    99→### Task ID: R6-1 — StatsCounter Bug Fix
   100→- **Bug**: The regex `value.replace(/[^0-9]/g, '')` extracted all digits from the string, so "1992-2026" became "19922026" (a 19 million count-up animation)
   101→- **Fix**: Changed to `/^[^\d]*(\d+)([^\d]*)$/` regex that only matches single integers. For multi-number values like "1992-2026", the match fails and the value is displayed statically without animation
   102→- **Verification**: VLM confirmed stats now correctly show "~15, 5000+, 1992-2026"
   103→- **Files Modified**: `/src/app/page.tsx`
   104→
   105→### Task ID: R6-2 — GameSetup CSS Bug Fix
   106→- **Bug**: `linear-gradient(135deg, #1a1a2e 0%, #22c55e/10 100%)` was invalid CSS — Tailwind's `color/opacity` syntax doesn't work in raw CSS gradients
   107→- **Fix**: Changed to `rgba(34, 197, 94, 0.1)` for proper opacity specification
   108→- **Also**: Simplified the duplicated border classes (`border-l-4 border-l-[#22c55e] border-t-... border-b-...` → `border-[#22c55e]`)
   109→- **Files Modified**: `/src/components/game/GameSetup.tsx`

  1536→### Critical Bug Found & Fixed: AnimatePresence `mode="wait"` broken
  1537→
  1538→**Symptom**: Clicking "Играть 30-0" button on homepage does NOT navigate to the Game Setup screen. The screen fades to black (opacity 0) but the new screen never appears.
  1539→
  1540→**Root Cause**: Framer Motion 12.26.2's `AnimatePresence mode="wait"` is broken with React 19. When the `key` prop changes (from "home" to "setup"), the exit animation fires and completes, but `onExitComplete` never triggers. As a result, AnimatePresence never mounts the new child component.
  1541→
  1542→**Debug Evidence**:
  1543→- `setScreen('setup')` IS called successfully (confirmed via `useGameStore.getState().screen` → "setup")
  1544→- The `Home` component DOES re-render with `screen: 'setup'` (confirmed via console.log)
  1545→- The exit animation fires and completes (`onAnimationComplete` fires for key "home")
  1546→- `AnimatePresence`'s internal `onExitComplete` NEVER fires
  1547→- The new child (key="setup") is NEVER mounted into the DOM
  1548→- DOM shows only one child: the exiting "home" screen at `opacity: 0; transform: translateX(-80px)`
  1549→
  1550→**Fix Applied**: Changed `AnimatePresence mode="wait"` to `AnimatePresence mode="popLayout"` with `layout` prop on the `motion.div`. This uses layout animations instead of sequential exit→enter, which works correctly with React 19.
  1551→
  1552→**File Changed**: `/home/z/my-project/src/app/page.tsx`
  1553→- Line 922: `mode="wait"` → `mode="popLayout"`
  1554→- Line 931: Added `layout` prop to `motion.div`
  1555→
  1556→**Note**: `SpinWheel.tsx` and `ManagerChoice.tsx` also use `mode="wait"` but with conditional rendering (not key changes), so they may still work. Should be tested separately.

## 1. Executive Summary

**30-0 RPL** — это мобильная игра-драфт для Telegram Mini Apps, созданная по образцу вирусного хита **38-0.app** (3.5M+ игроков), но адаптированная под **российский футбольный чемпионат** (Высшая лига / РФПЛ / РПЛ) с 1990 по 2026 год.

### Ключевая идея
Игрок собирает **идеальную сборную всех времён** из игроков российского чемпионата, «крутит колесо» для получения случайного клуба и сезона, выбирает одного игрока, и повторяет, пока не заполнит 11 позиций. Затем симулируется полный сезон (30 матчей в РПЛ) — цель достичь **30-0** (30 побед, 0 ничьих, 0 поражений, 90 очков).

### Формула успеха (из 38-0)
- **Простая механика**: Спин → Выбор → Назначение → Повтор
- **Эмоциональный крючок**: «Ещё одна попытка, вдруг получится 30-0!»
- **Ностальгия**: Игроки из разных эпох российского футбола
- **Виральность**: Шерить результат в Telegram естественно
- **Бесплатно**: Без доната, без рекламы (на старте)

### Что реализовано в MVP ✅
- ✅ **Классический режим драфта** (Classic Draft)
- ✅ Выбор формации (12 схем) с типами (⚔️ Атака, 🛡️ Защита, ⚖️ Баланс, 🎯 Полузащита)
- ✅ Выбор сложности (Easy/Normal/Hard) с перебросами
- ✅ Спин колеса → выбор игрока → назначение позиции
- ✅ Симуляция сезона (30 матчей с турнирной таблицей)
- ✅ Профиль со статистикой (винрейт, трофеи, история)
- ✅ Лидерборд (глобальный)
- ✅ Быстрый старт (⚡ Быстрый старт — случайные настройки)

### 2.4 Уникальные правила драфта

1. **Одноимённый игрок = один раз**: Если выбран «Алан Дзагоев», его нельзя выбрать снова ни из другого клуба, ни из другого сезона
2. **Колесо умное**: Спин приземляется ТОЛЬКО на клубы, у которых есть игроки на оставшиеся свободные позиции
3. **Совместимость позиций**: Игрока можно поставить на позицию, совместимую с его основной (напр., CAM может играть CM, но не CB)
4. **Штраф за несовместимость**: Если поставить игрока «не на позицию», его рейтинг умножается на 0.8
5. **Переброс (Reroll)**: Можно скинуть выпавший клуб-сезон и крутить снова (количество зависит от сложности)

## 4. Игровая механика (Детальный разбор)

### 4.1 Колесо (Spin Wheel)

**Как работает**:
1. Пользователь нажимает «Крутить»
2. Анимация вращения колеса (~2 сек)
3. Колесо останавливается на **случайном клубе + сезоне**
4. Важно: колесо приземляется **только на клубы, у которых есть игроки на свободные позиции**

**Фильтрация клуба**:
- Получить все свободные позиции в формации
- Определить, какие позиции могут их заполнить
- Из пула клуб-сезонов оставить только те, у которых есть хотя бы один игрок на совместимую позицию
- Из отфильтрованного пула выбрать случайный

**Вероятность**: Все клуб-сезоны равновероятны (нет утяжеления)

### 4.2 Выбор игрока

После спина показан список игроков выпавшего клуба-сезона:
- **Имя**: Фамилия (на русском)
- **Рейтинг**: Число 40-99 (зависит от сезона и выступления)
- **Позиции**: Одна основная + совместимые (напр., ЦП, ОП, АП)
- **Цвет рейтинга**: Зелёный (78+), Голубой (73-77), Оранжевый (68-72), Красный (<68)

**Правило уникальности**: Один и тот же человек (по ФИО) не может быть выбран дважды, даже из другого сезона/клуба.

### 4.8 Симуляция сезона

**Алгоритм** (упрощённый, на основе 38-0):

1. **Расчёт силы команды** по линейным позициям:
   - `ATT` = средний рейтинг нападающих
   - `MID` = средний рейтинг полузащитников
   - `DEF` = средний рейтинг защитников
   - `GK` = рейтинг вратаря
   - `OVERALL` = взвешенное среднее (ATT×0.3 + MID×0.3 + DEF×0.3 + GK×0.1)

2. **Генерация соперников**: 15 команд × 2 матча (дом/выезд)
   - Сила соперника = случайная из пула исторических рейтингов клубов РПЛ (55-85)
   - Домашний бонус: +3 к рейтингу хозяев

3. **Расчёт каждого матча**:
   ```
   разница = (OVERALL_команда + бонус_дома) - рейтинг_соперника
   P(победа) = sigmoid(разница / 10)
   P(ничья) = (1 - P(победа)) × 0.3 × (1 - abs(разница)/50)
   P(поражение) = 1 - P(победа) - P(ничья)
   ```

4. **Голы**:
   - Ожидаемые голы = base × 10^(ATT/100)
   - Фактические голы = Poisson(xG)
   - Пропущенные = Poisson(base × 10^(соперник_ATT/100) × (1 - DEF/200))

5. **Бонус тренера**: +2 к overall при наличии тренера

6. **Вариативность**: Тот же состав может дать разные результаты при пересимуляции

### 4.9 Достижения (Трофеи)

| Трофей | Условие |
|--------|---------|
| 30-0 | Выиграть все 30 матчей |
| Непобедимый | 0 поражений за сезон |
| Чемпион | Занять 1 место |
| Топ-3 | Занять 1-3 место |
|Century голов | Забить 100+ голов |
| Железная защита | Пропустить ≤10 голов |
| Крепость | Пропустить ≤20 голов |
| Все отечественные | Состав только из россиян |
| Одна команда | Все игроки из одного клуба |

### 7.1 Prisma Schema

```prisma
// === ПОЛЬЗОВАТЕЛИ ===
model User {
  id            String   @id // Telegram user ID
  username      String?
  firstName     String?
  lastName      String?
  photoUrl      String?
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
  
  runs          Run[]
  achievements  UserAchievement[]
  
  favouriteFormation String?
  totalSeasons   Int     @default(0)
  bestPoints     Int     @default(0)
  bestRecord     String? // "27-2-1"
  titles         Int     @default(0)
  perfectSeasons Int     @default(0)
}

// === КЛУБЫ ===
model Club {
  id          String   @id @default(cuid())
  nameRu      String
  nameEn      String?
  city        String?
  logoUrl     String?
  founded     Int?
  
  seasons     ClubSeason[]
  
  @@unique([nameRu])
}

// === СЕЗОНЫ ===
model Season {
  id          String   @id @default(cuid())
  startYear   Int      // 1990, 1991, ...
  endYear     Int      // 1991, 1992, ...
  label       String   // "1990", "2011/12"
  matchesPerTeam Int   // 30, 38, 44
  
  clubSeasons ClubSeason[]
  
  @@unique([startYear, endYear])
}

// === КЛУБ В СЕЗОНЕ ===
model ClubSeason {
  id          String   @id @default(cuid())
  clubId      String
  seasonId    String
  position    Int?     // Итоговое место в таблице
  points      Int?     // Очки
  played      Int?     // Матчей
  won         Int?
  drawn       Int?
  lost        Int?
  goalsFor    Int?
  goalsAgainst Int?
  
  club        Club     @relation(fields: [clubId], references: [id])
  season      Season   @relation(fields: [seasonId], references: [id])
  players     PlayerSeason[]
  
  @@unique([clubId, seasonId])
}

// === ИГРОКИ ===
model Player {
  id          String   @id @default(cuid())
  lastName    String   // Фамилия
  firstName   String?  // Имя (опционально)
  fullName    String   // Полное ФИО для уникальности
  nationality String?  // Россия, Украина, и т.д.
  birthYear   Int?
  photoUrl    String?
  
  seasons     PlayerSeason[]
  
  @@unique([fullName])
}

// === ИГРОК В СЕЗОНЕ ===
model PlayerSeason {
  id            String   @id @default(cuid())
  playerId      String
  clubSeasonId  String
  rating        Int      // 40-99, основной рейтинг
  mainPosition  String   // "ЦП", "НП", "ВР", и т.д.
  otherPositions String? // "ЦП,ОП,АП" через запятую
  age           Int?
  matches       Int?
  goals         Int?
  assists       Int?
  
  player        Player     @relation(fields: [playerId], references: [id])
  clubSeason    ClubSeason @relation(fields: [clubSeasonId], references: [id])
  runSlots      RunPlayer[]
  
  @@unique([playerId, clubSeasonId])
}

// === ФОРМАЦИИ ===
model Formation {
  id          String   @id @default(cuid())
  name        String   // "4-3-3"
  slots       String   // JSON: [{"pos":"ВР","cat":"gk"},{"pos":"ЦЗ","cat":"def"},...]
  
  runs        Run[]
}

// === ЗАПУСК (ПОПЫТКА) ===
model Run {
  id            String   @id @default(cuid())
  userId        String
  formationId   String
```

### 10.2 Стратегия сбора данных

**Этап 1 (MVP)**: Ручной ввод топ-клубов × ключевых сезонов
- 10 топ-клубов × 5-6 сезонов = ~50 клуб-сезонов
- ~10-15 игроков на клуб-сезон = ~600-750 PlayerSeason
- Время: ~20-30 часов ручной работы

**Этап 2 (Расширение)**: Парсинг Transfermarkt
- Скрипт на Python для скрапинга составов
- Нормализация имён и позиций
- Автоматическая генерация рейтингов

**Этап 3 (Полная база)**: Все 36 сезонов × все клубы
- Целевой объём: 5000+ PlayerSeason
- Проверка и нормализация

### 10.3 Методика рейтингов

Рейтинг игрока (40-99) на конкретный сезон рассчитывается как:

```
baseRating = 50 + (minutesPlayed / totalPossibleMinutes) × 20
           + (goals / matches) × 5 (для нападающих)
           + (assists / matches) × 3
           + (clubFinalPosition_bonus: 1-е место +5, 2-е +3, 3-е +2)
           - (age_factor: если > 34, -3; если < 20, -2)

finalRating = clamp(baseRating, 40, 99)
```

Для калибровки можно использовать FIFA-рейтинги как ориентир:
- 90+ = Легенда (Аршавин 2008, Акинфеев 2006)
- 80-89 = Звезда (Дзагоев 2012, Зырянов 2008)
- 70-79 = Основной игрок
- 60-69 = Запасной
- < 60 = Редко играет

## Приложение В: Ключевые решения и компромиссы

| Решение | Обоснование | Альтернатива |
|---------|-------------|--------------|
| 30 матчей вместо переменного числа | Упрощение, брендинг «30-0» | Историческая точность (30/34/38/44) |
| SQLite вместо PostgreSQL | Простота MVP, нет хостинга БД | PostgreSQL для продакшена |
| Нет регистрации без Telegram | Целевая платформа = Telegram Mini App | Email/Google auth |
| Ручной ввод данных на старте | Быстрее запустить MVP | Полный парсинг сразу |
| Framer Motion для анимаций | Красиво и просто | CSS animations |
| Одностраничное приложение | Все экраны на `/` через Zustand state | Многостраничное с роутами |
| Canvas для колеса вместо SVG | Лучшая производительность при анимации | SVG с CSS transforms |
| Zustand persist для драфта | Продолжение после закрытия вкладки | Только серверное состояние |
| goHome() vs resetGame() | goHome сохраняет состояние драфта для resumeGame | Один метод с флагом |
| btn-inner-shimmer CSS анимация | Визуальная обратная связь на CTA | hover:scale-105 (убран) |
